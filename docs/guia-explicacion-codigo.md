# Guia para explicar el codigo

Este documento indica como explicar el proyecto en futuras sesiones.

## Estilo de explicacion

- Explicar de manera simple, para una persona con conocimientos basicos de HTML, CSS y JavaScript.
- Evitar lenguaje demasiado tecnico.
- Si aparece una palabra tecnica, explicarla con un ejemplo simple.
- Explicar por bloques pequenos cuando el codigo sea largo.
- Indicar que archivo se modifica y por que.

## Que explicar de cada archivo

- Para que sirve.
- Que responsabilidad tiene.
- Como se relaciona con otros archivos.
- Que parte deberia tocar la persona si quiere cambiar algo.

## Cuando se agregue una funcionalidad

Explicar:

- Que problema resuelve.
- Que archivos se modificaron.
- Que codigo se agrego.
- Como probar que funciona.

## Conceptos importantes

- HTML semantico: usar etiquetas como `header`, `main`, `section` y `footer` para que la pagina tenga una estructura clara.
- CSS variables: nombres reutilizables para colores, tipografias y espacios. Se definen en `:root`.
- Responsive design: diseno que se adapta a celular, tablet y escritorio.
- Array de productos: lista de productos guardada en JavaScript.
- Objeto de JavaScript: grupo de datos de un producto, por ejemplo nombre, precio e imagenes.
- Funcion de JavaScript: bloque de codigo que hace una tarea.
- Evento click: accion que ocurre cuando una persona hace click.
- Redireccion a WhatsApp: enlace que abre WhatsApp con un mensaje preparado.
- Media query: regla CSS que cambia estilos segun el ancho de pantalla.
- Renderizado dinamico: crear contenido desde JavaScript en lugar de escribirlo a mano en HTML.
- Filtro por categoria: mostrar solo productos de una categoria elegida.
- Galeria de imagenes: futura vista para mostrar todas las fotos de un producto.
- Ruta de imagen: ubicacion del archivo, por ejemplo `assets/img/agendas/agenda-club-1.png`.
