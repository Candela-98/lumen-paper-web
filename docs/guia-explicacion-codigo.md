# Guía para explicar el código

Este documento indica cómo explicar el proyecto en futuras sesiones.

## Estilo de explicación

- Explicar de manera simple, para una persona con conocimientos básicos de HTML, CSS y JavaScript.
- Evitar lenguaje demasiado técnico.
- Si aparece una palabra técnica, explicarla con un ejemplo simple.
- Explicar por bloques pequeños cuando el código sea largo.
- Indicar qué archivo se modifica y por qué.

## Qué explicar de cada archivo

- Para qué sirve.
- Qué responsabilidad tiene.
- Cómo se relaciona con otros archivos.
- Qué parte debería tocar la persona si quiere cambiar algo.

## Cuando se agregue una funcionalidad

Explicar:

- Qué problema resuelve.
- Qué archivos se modificaron.
- Qué código se agregó.
- Cómo probar que funciona.

## Conceptos importantes

- HTML semántico: usar etiquetas como `header`, `main`, `section` y `footer` para que la página tenga una estructura clara.
- CSS variables: nombres reutilizables para colores, tipografías y espacios. Se definen en `:root`.
- Responsive design: diseño que se adapta a celular, tablet y escritorio.
- Array de productos: lista de productos guardada en JavaScript.
- Objeto de JavaScript: grupo de datos de un producto, por ejemplo nombre, precio e imágenes.
- Función de JavaScript: bloque de código que hace una tarea.
- Evento click: acción que ocurre cuando una persona hace click.
- Redirección a WhatsApp: enlace que abre WhatsApp con un mensaje preparado.
- Media query: regla CSS que cambia estilos según el ancho de pantalla.
- Renderizado dinámico: crear contenido desde JavaScript en lugar de escribirlo a mano en HTML.
- Filtro por categoría: mostrar solo productos de una categoría elegida.
- Galería de imágenes: vista que muestra una imagen grande y miniaturas de un producto.
- Modal: ventana interna de la página que muestra más información sin abrir una pestaña nueva.
- Botón flotante: enlace fijo que queda visible aunque la persona haga scroll.
- Sección de pasos: bloque de contenido que explica un proceso en cards ordenadas.
- Favicon: icono pequeño que aparece en la pestaña del navegador.
- SEO básico: textos y etiquetas que ayudan a los buscadores a entender la página.
- Open Graph: etiquetas que ayudan a que el sitio se vea mejor cuando se comparte en redes o WhatsApp.
- `event.stopPropagation()`: instrucción que frena el click para que no active también el evento de un elemento padre.
- Ruta de imagen: ubicación del archivo, por ejemplo `assets/img/agendas/agenda-club-1.png`.
